HTTP Handlers

Reference documentation for all HTTP request handlers.

Handler Overview

HTTP handlers process incoming requests and return responses. Each handler is responsible for a specific resource or collection of related endpoints.

Core Handlers

cb_health_handler
Returns system health status. Used for load balancer health checks and monitoring.
MethodPathDescription
GET/healthReturns health status
cb_not_found_handler
Handles unmatched routes. Returns 404 error with documentation link.

Party Handlers

cb_parties_handler
Handles party (customer) collection operations.
MethodPathDescription
GET/partiesList all parties
POST/partiesCreate new party
cb_party_handler
Handles individual party operations.
MethodPathDescription
GET/parties/:idGet party by ID
PATCH/parties/:idUpdate party
DELETE/parties/:idClose party

Handler Response Format

% Success response (200/201)
{ok, Reply, Req} ->
    {true, #{<<"data">> => cb_schema:serialize(Reply)}, Req};

% Error response (4xx/5xx)
{error, Reason} ->
    Error = cb_http_errors:format_error(Reason),
    {Error, Req}.
