IronLedger Documentation

A Dockerized core banking system built with Erlang/OTP, featuring double-entry bookkeeping, product management, loans, and a Go WebAssembly dashboard.

System Overview

IronLedger is an Erlang/OTP 25.3 core banking application that provides:

  • Account Management - Create, freeze, unfreeze, and close bank accounts
  • Double-Entry Ledger - Every transaction maintains the accounting equation (Assets = Liabilities + Equity)
  • Payment Processing - Transfers, deposits, withdrawals with idempotency guarantees
  • Loan Management - Loan products, disbursements, repayments, and interest calculations
  • Savings Products - Interest-bearing deposit accounts with flexible compounding
  • REST API - HTTP API via Cowboy with JSON responses
  • Wasm Dashboard - Browser-based UI compiled from Go to WebAssembly with locally packaged fonts and Material Symbols

Core Principles

Monetary Arithmetic

All amounts are stored as non-negative integers in minor units (cents, pence, etc.). For example, $100.00 USD is stored as 10000. This prevents floating-point precision errors critical in financial systems.

Double-Entry Bookkeeping

Every financial transaction creates paired debit and credit entries. The sum of all debits always equals the sum of all credits, maintaining the fundamental accounting equation.

Idempotent Operations

All payment operations accept an idempotency key (client-generated UUID) that prevents duplicate processing when operations are retried due to network failures.

Immutable Ledger

Once posted, ledger entries cannot be modified or deleted. Corrections are made by posting new reversal or adjustment entries, creating a complete audit trail.

Data Storage

All financial data is stored in Mnesia (Erlang's distributed DBMS):

  • RAM + Disk - Tables use both RAM and disk storage for performance and durability
  • Transactions - All financial operations use mnesia:transaction/1 for atomicity
  • No Dirty Operations - Financial tables never use dirty reads/writes
